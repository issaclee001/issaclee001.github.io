jacoco代码覆盖率统计

  • 测试

##软件安装
前提在服务所在服务器

  • Apache Ant
    下载linux安装包，apache-ant-1.9.9-bin.tar.gz
    解压到：/export/apache-ant(没有自行创建)
    配置环境变量：
    export ANT_HOME=/export/apache-ant/apache-ant-1.9.9
    export PATH=$PATH:$ANT_HOME/bin
    最后ant –v,保证ant安装成功
  • Jacoco安装步骤
    下载最新版本Jacoco压缩包（http://www.jacoco.org/jacoco/）
    解压并上传jacocoagent.jar和jacocoant.jar，如：/export/jacoco（若无则自行创建）
  • 创建目录
    /export/jacoco/result —–存放扫描结果
    /export/jacoco/report ——存放最终报告
    /export/jacoco/source ——存放想要扫描的源代码文件（为了报告准确，尽量精准）
    /export/jacoco/class ——存放想要扫描的源代码对应的class文件

配置tomcat（如若检测的是service或worker应用，则跳过此步）

  • 进入tomcat目录，如：/export/servers/test/env_1/rcc/bin
  • 关闭tomcat服务
  • 修改 catalina.sh 的JAVA_OPTS配置(具体参数的意思请查看官方文档)
    JAVA_OPTS=”$JAVA_OPTS -javaagent:/export/jacoco/jacocoagent.jar=includes=*,output=file,append=false,destfile=/export/jacoco/result/jacoco.exec”
  • 启动服务 ./startup.sh
  • ps查看一下JAVA_OPTS是否有修改正确
  • 执行jenkins 自动化job，获取覆盖率数据
  • 执行完以后关闭服务(kill 进程号ID，不要使用kill -9 )，到此在/export/jacoco/result/目录下，会有一个文件大小大于0以jacoco.exec文件

配置service(worker类似)

  • 进入service目录，如：/export/data/tomcatRoot/test/Store_api/bin
  • 关闭服务
  • 修改start.sh脚本
  • 启动服务，sh start.sh
  • ps查看一下JAVA_OPTS是否有修改正确
  • 执行jenkins 自动化job，获取覆盖率数据
  • 执行完以后关闭服务(kill 进程号ID，不要使用kill -9 )，到此在/export/jacoco/result/目录下，会有一个文件大小大于0以jacoco.exec文件

创建并配置build.xml

路径：/export/apache-ant/apache-ant-1.9.9/

执行ant report

进入/export/jacoco/report，并将该目录下所有文件下载到本地，最后查看生成的报告

另：目前的方式比较笨拙，需要人工操作的地方太多，后续会将整套集成一下，形成更好的解决方案
